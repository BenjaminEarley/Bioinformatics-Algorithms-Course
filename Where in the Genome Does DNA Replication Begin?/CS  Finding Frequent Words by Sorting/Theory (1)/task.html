Since identical k-mers clump together in the sorted array (like (0, 0, 0) for AA or (10, 10, 10) for GG), frequent k-mers are the longest runs of identical integers in SortedIndex.

This insight leads to the FindingFrequentWordsBySorting algorithm, shown below. This algorithm uses an array Count for which Count(i) computes the number of times that the integer at position i in the array SortedIndex appears in the first i elements of this array (shown at the bottom of the above figure). In the pseudocode for FindingFrequentWordsBySorting, we assume that you already know how to sort an array using a function Sort.

    FindingFrequentWordsBySorting(Text , k)
        FrequentPatterns ← an empty set
        for i ← 0 to |Text| − k
            Pattern ← Text(i, k)
            Index(i) ← PatternToNumber(Pattern)
            Count(i) ← 1
        SortedIndex ← Sort(Index)
        for i ← 1 to |Text| − k
            if SortedIndex(i) = SortedIndex(i − 1)
                Count(i) = Count(i − 1) + 1
        maxCount ← maximum value in the array Count
        for i ← 0 to |Text| − k
            if Count(i) = maxCount
                Pattern ← NumberToPattern(SortedIndex(i), k)
                add Pattern to the set FrequentPatterns
        return FrequentPatterns
